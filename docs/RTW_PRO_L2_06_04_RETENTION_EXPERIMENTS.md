## 목적
- 기능 추가 전 운영 실험으로 재참여율을 개선한다.

## 실험 가설
- 정기 이벤트 시간 최적화가 재참여율을 높인다.
- 결과 화면 내 재도전 CTA 강화가 24시간 내 재참가율을 높인다.
- 이벤트 사전 알림 타이밍 최적화가 방 입장 성공률을 높인다.

## 실험 설계
- 실험 단위: 주간(최소 1주), 실험군/대조군 분리
- 기본 샘플: 각 그룹 최소 30명
- 실험 중 변경 금지: 레이스 핵심 규칙/순위 계산 로직

## 측정 지표
- D1 재참가율, D7 재참여율
- 이벤트 알림 클릭률
- 방 입장 성공률
- 사용자 불만 신고율(알림 과다/이벤트 피로)

## 운영 규칙
- 동일 사용자에게 2개 이상의 실험을 동시 적용하지 않는다.
- 부정적 영향(불만 신고율 급증)이 감지되면 즉시 롤백한다.
- 실험 결과는 KPI 대시보드 주간 리포트에 함께 기록한다.

## 완료 기준
- 베타 기간 내 최소 2개의 재참여 실험이 완료된다.
- 유의미한 개선안 1개 이상이 정식 출시 운영안으로 승격된다.
