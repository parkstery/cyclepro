## UX 원칙
- 스마트폰 화면 기준으로 주행 중 정보량을 최소화한다.
- 거리뷰는 몰입 배경 역할로 유지한다.
- 경쟁 정보는 HUD/미니맵/이벤트 레이어로 전달한다.
- 화려한 3D 대신 즉각 피드백과 비교 체감을 우선한다.
- 주행 중 단순함과 주행 후 분석 깊이를 분리해 Pro 학습 경험을 만든다.

## 정보 우선순위
- `[CORE]` 순위, 앞/뒤 갭, 남은 거리, 현재 속도
- `[AUX]` 평균속도, 고도 그래프, 상세 통계
- `[POST]` 결과 분석, 리플레이, 비교 카드, 추월 타임라인, 구간 페이스 편차

## 소셜 이벤트
- 라이더 입장/퇴장 알림
- 카운트다운 시작/출발 알림
- 추월/피추월 알림
- 간단 퀵 리액션(응원/파이팅)

## 정기 이벤트 템플릿
- 예: `매일 20:00 한강 10km`
- 방장 템플릿 저장 후 반복 개설
- 참여자에게 사전 알림(푸시/인앱)

## 완료 기준
- “같이 달린다” 체감을 만드는 우선순위 기능이 확정된다.
- 주행 중 화면은 CORE 중심으로 단순하게 유지된다.
- 주행 후 분석 화면에서 "왜 이겼고 왜 졌는지"를 설명할 수 있는 최소 인사이트 세트가 확정된다.
